# Documentation of Changes Made to Program

### March 15, 2026

I modularized the codes so we can have a clearer understanding of the 
responsibility of each file (commands, UI, background tasks, setting up the bot, etc)

Why? 
- This makes for easier debugging. We can find and fix issues faster if the code is organized neatly.
- Scaling is easier. Adding new features and commands will be easier in the long run.
- We can avoid circular import and make dependencies clear.
